# Information Processing & Management — Elsevier Submission Guidelines

---

## 1. Writing and Formatting

### File Format
- Provide **editable source files** for the entire submission (text, figures, tables, graphics).
- Word files: `.doc` / `.docx`
- LaTeX files: `.tex`
- **PDF is not acceptable** as a source file.
- Word files must use **single-column layout**. Double-column is permitted for LaTeX only.
- **Remove all strikethrough and underlined text** unless it carries scientific significance.
- Run **spell-check and grammar-check** before submission.

---

## 2. Title Page

The title page must include:

| Element | Requirements |
|---|---|
| **Article Title** | Concise and informative; avoid abbreviations/formulae unless widely established (e.g., DNA) |
| **Author Names** | Given name(s) + family name(s); match the order in the submission system; check spelling carefully |
| **Affiliations** | Full postal address of each affiliation including country; email address if available; use lowercase superscript letters |
| **Corresponding Author** | Clearly indicated; must include up-to-date email and contact details |
| **Present/Permanent Address** | Use superscript Arabic numerals for footnotes if an author has moved or was visiting |

---

## 3. Double Anonymized Peer Review

This journal uses a **double anonymized review process** — author identities are concealed from reviewers and vice versa.

Submit **two separate files**:

### Title Page File (with author details)
In addition to the items above, include:
- Acknowledgements (language help, writing, proofreading, etc.)
- Declaration of competing interests (if no separate declaration file is submitted)
- Corresponding author's full address
- Corresponding author's email address

> ⚠️ Do **not** add acknowledgements as a footnote to the title or elsewhere in the article.

### Anonymized Manuscript File (without author details)
- Contains the main body, references, and tables
- Must **not** include author names, affiliations, or acknowledgements

---

## 4. Abstract

- Maximum **250 words**
- Must be able to **stand alone** (abstracts are often presented separately)
- Briefly state: purpose, principal results, major conclusions
- **Avoid references** where possible; if essential, cite author(s) and year(s)
- **Avoid non-standard abbreviations**; if essential, define at first mention

---

## 5. Keywords

- Provide **1–7 keywords** in English
- Avoid multi-word phrases using "and" or "of" where possible
- Use abbreviations only if firmly established in the field

---

## 6. Tables

- Submit as **editable text**, not images
- Place next to relevant text or on a separate page at the end
- **Cite all tables** in the manuscript
- Number tables **consecutively** in order of appearance
- Include **captions** for all tables
- Place **table notes below** the table body
- **Avoid vertical rules and shading** within cells
- Do not duplicate results described elsewhere in the article

---

## 7. Figures, Images, and Artwork

- Submit each figure as a **separate file**
- Use a logical naming convention (e.g., `Figure_1`, `Figure_2`)
- **Cite all images** in the manuscript text
- Number images in order of appearance
- Text graphics may be embedded in the text (LaTeX: also in the file)

### Captions
- Every figure must have a caption
- Caption = brief title + description of the image
- Keep text within the image to a minimum; explain all symbols/abbreviations in the caption

### File Formats and Resolution

| Artwork Type | Format | Minimum Resolution |
|---|---|---|
| Vector drawings | EPS or PDF (embed font or save text as graphics) | — |
| Color/grayscale photographs (halftones) | TIFF, JPG, or PNG | 300 dpi (single col: ≥1,063 px; full page: ≥2,244 px) |
| Bitmapped line drawings | TIFF, JPG, or PNG | 1,000 dpi (single col: ≥3,543 px; full page: ≥7,480 px) |
| Combinations (line + halftone) | TIFF, JPG, or PNG | 500 dpi (single col: ≥1,772 px; full page: ≥3,740 px) |

### Do NOT Submit
- Files too low in resolution
- Artwork with disproportionately small text
- Multiple images/graphs combined into a single file

### Color Artwork
- Accepted color figures will appear in **color online**
- Ensure color images are **accessible** to those with impaired color vision
- For print color, cost information will be provided after acceptance

---

## 8. Research Data

- **Deposit** research data in a relevant data repository (Option C applies)
- **Cite and link** to the dataset in the article
- If sharing is not possible, include a **statement explaining why**

Research data includes: results of observations/experiments, software, code, models, algorithms, protocols, methods, and other relevant materials.

---

## 9. Article Structure

Divide the manuscript into clearly defined sections with headings covering all essential elements.

### Author Contributions: CRediT

Corresponding authors must acknowledge co-author contributions using [CRediT (Contributor Roles Taxonomy)](https://credit.niso.org/):

- Conceptualization
- Data curation
- Formal analysis
- Funding acquisition
- Investigation
- Methodology
- Project administration
- Resources
- Software
- Supervision
- Validation
- Visualization
- Writing – original draft
- Writing – review and editing

> Not all roles apply to every manuscript; authors may contribute through multiple roles.

### Appendices

- Label appendices as **A, B, C, …**
- Number equations as **Eq. (A.1), Eq. (A.2)**, etc.; subsequent appendices as **Eq. (B.1)**, etc.
- Number tables and figures as **Table A.1**, **Fig. A.1**, etc.

---

## 10. References

### General Rules
- All in-text citations must appear in the reference list and vice versa
- References cited in the abstract must be given **in full**
- Avoid citing unpublished results and personal communications in the reference list (mention in text only)
- If included, follow journal reference style; use "unpublished results" or "personal communication" in place of a date
- "In press" = accepted for publication
- Use **DOIs** as reference links wherever available

### Web References
At minimum include:
- Full URL
- Date last accessed
- Additional info (DOI, author names, dates, source publication) if known

Can be listed separately after the reference list or integrated within it.

### Data References
Include:
- Author name(s)
- Dataset title
- Data repository
- Version (where available)
- Year
- Global persistent identifier

> Prefix data references with **[dataset]** — this identifier will not appear in the published article.

### Preprint References
- Clearly mark preprints (include "preprint" or the server name, and the preprint DOI)
- If a peer-reviewed version is available, **use the formal publication** as the reference
- Unformally published preprints central to the work may still be referenced

### Reference Management Software
- Use [Mendeley Reference Manager](https://www.mendeley.com/reference-management/reference-manager/) or any CSL-compatible tool with the journal template
- **Remove all field codes** before submission
